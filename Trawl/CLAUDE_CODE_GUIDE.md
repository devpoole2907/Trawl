## Hard Constraints — Do NOT Violate

1. **iOS 26 only.** No `#available` checks. Assume iOS 26 APIs exist unconditionally.
2. **`@Observable` only.** Never use `ObservableObject`, `@Published`, `@StateObject`, `@ObservedObject`.
3. **No Combine.** Zero `import Combine`.
4. **No DispatchQueue.** All concurrency via `async/await`, `Actor`, `Task`.
5. **SwiftData for persistence.** UserDefaults only for trivial scalar prefs.
6. **Keychain for secrets.** Credentials never touch SwiftData or UserDefaults.
7. **No UIKit in views** except DocumentPicker and ShareViewController.

## Integration Steps

### 1. Create the Xcode Project
- New → App → SwiftUI, Swift, SwiftData
- Product Name: `Trawl`
- Bundle ID: `com.jamesoberle.Trawl`
- Deployment target: iOS 26.0

### 2. Add All Source Files
- Drag the entire directory structure into the Xcode project navigator
- Ensure all `.swift` files are added to the `Trawl` target
- Exception: `ShareExtension/` files go in the `TrawlShare` target (step 4)

### 3. Configure Entitlements
- In the Trawl target → Signing & Capabilities:
  - Add "App Groups" → `group.com.jamesoberle.Trawl`
  - Add "Keychain Sharing" → `com.jamesoberle.Trawl.shared`
  - Add "Background Modes" → enable "Background fetch" and "Background processing"
  - Add "Time Sensitive Notifications"
- Set the entitlements file to `Trawl.entitlements`

### 4. Create the Share Extension Target
- File → New → Target → Share Extension
- Product Name: `TrawlShare`
- Delete the auto-generated `ShareViewController.swift` (we have our own)
- Add these files to BOTH targets (Trawl + TrawlShare):
  - `Utilities/AppGroup.swift`
  - `Utilities/KeychainHelper.swift`
  - `Models/ServerProfile.swift`
  - `Models/RecentSavePath.swift`
  - `Models/CachedTorrentState.swift`
  - `Models/API/` (all files)
  - `Services/QBError.swift`
  - `Services/AuthService.swift`
  - `Services/QBittorrentAPIClient.swift`
- Add Share Extension-only files to TrawlShare target:
  - `ShareExtension/ShareViewController.swift`
  - `ShareExtension/ShareAddTorrentView.swift`
- Configure TrawlShare entitlements:
  - App Groups: `group.com.jamesoberle.Trawl`
  - Keychain Sharing: `com.jamesoberle.Trawl.shared`
- Set `ShareExtension/Info.plist` as the Info.plist for TrawlShare
- Set `ShareExtension/TrawlShare.entitlements` as entitlements file

### 5. Add Background Modes to Main App Info.plist
```xml
<key>UIBackgroundModes</key>
<array>
    <string>fetch</string>
    <string>processing</string>
</array>
```

### 6. Verify and Fix Compilation

Common issues to watch for:

- **Sendable conformance**: `SyncService` and `TorrentService` are `@Observable` classes
  used with `.environment()`. If Swift 6 strict concurrency flags issues, you may need
  `@MainActor` annotations on the `@Observable` classes or `nonisolated(unsafe)` on
  specific properties. Start by adding `@MainActor` to `SyncService`, `TorrentService`,
  `AppServices`, and all ViewModels if needed.

- **`@Bindable` usage**: In views where we use `@Bindable var vm = vm`, ensure the
  local binding pattern works. If not, switch to `@State` ownership or pass bindings
  explicitly.

- **`Torrent` mutability**: `Torrent` fields are `var` so `applying(delta:)` can create
  modified copies. If Codable complains, fields referenced only via CodingKeys can stay `let`.

- **`TorrentFile` custom decoding**: The `init(from:)` sets `index = -1` as a sentinel.
  The `withIndex(_:)` method creates the properly-indexed copy. Make sure
  `QBittorrentAPIClient.getTorrentFiles` calls `.withIndex()` during enumeration.

- **Environment injection**: `SyncService` and `TorrentService` are injected via
  `.environment()` in `ContentView`. Every view that reads them needs
  `@Environment(SyncService.self)` and `@Environment(TorrentService.self)`.

### 7. Test Priority Order

1. **Onboarding flow**: Add a server, verify Keychain storage
2. **Sync loop**: Connect to qBittorrent, verify torrents appear and update in real time
3. **Add torrent**: Test magnet link and .torrent file paths
4. **Detail view**: Verify properties load, file list renders, actions work
5. **Swipe actions**: Pause/resume/delete from list
6. **Settings**: Polling interval change, notification toggle
7. **Share Extension**: Share a magnet link from Safari
8. **Background notifications**: Use Xcode "Simulate Background Fetch"

### 8. Liquid Glass Polish (Post-Functional)

Once everything works, add iOS 26 Liquid Glass polish:
- `.glassEffect()` on toolbar backgrounds, card sections, filter chips
- `.containerBackground()` for sheet presentations  
- Material backgrounds on detail view sections (already using `.regularMaterial`)
- Translucent tab bars and navigation bars
- `.presentationBackground(.ultraThinMaterial)` on sheets

## Known Edge Cases

1. **Self-signed certs**: Many qBittorrent instances use self-signed HTTPS. Consider adding
   a `URLSessionDelegate` with a user toggle in Settings. For V1, document the limitation.

2. **SID expiry**: Handled by 403 retry logic in `QBittorrentAPIClient.performRequest`.

3. **Large torrent lists**: Start with `List`, optimise to `LazyVStack` if needed.

4. **First sync response**: `rid=0` returns full dataset. Can be large. Parsed in one shot.

5. **Empty deltas**: When nothing changes, `/sync/maindata` returns `{"rid": N}` only.
   `applyDelta` handles all-nil fields gracefully.

6. **TorrentFile index**: API doesn't return index — array position IS the index.
   Handled in `getTorrentFiles` with `.enumerated().map { withIndex($0) }`.
